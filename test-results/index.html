SVG Accessibility Test Matrix

This document compares the capabilities of SVG in popular screen reader and browser combinations. The tests represented cover basic SVG support with screen readers (a kind of Assistive Technology or AT), and comparison and contrast with ARIA behavior in SVG vs equivalent structures in HTML.

The goal is to identify instances where SVG differs from HTML, or from the behavior specified in the relevant SVG or WAI-ARIA specifications. Once issues are identified, we can determine whether the problem originates in the screen reader, the browser, the operating systems accessibility APIs, or some combination.

Once the issue and the responsible software is identified, we can work to get that behavior changed.

In some cases, the behavior may work correctly as directed in the relevant specification, but may lead to suboptimal authoring or reading experiences. In other cases, the behavior may be underspecified, and thus need better specification to provide interoperable authoring and reading experiences across all screen reader, browser, and operating systems.

Technical Note

The tables in this document are dynamically generated from a JSON file, svg-a11y-tests.json, which is a summary of the detailed issue reports. This JSON file is intended to provide up-to-date test results in a reader-friendly presentation, but requires that client-side JavaScript is enabled.

Implementations

In order to sample the most common and representative screen-reader, browser, and operating system combinations, we referenced the 2021 WebAim Screen Reader Survey. We selected a subset of combinations that balanced the reported frequency of combinations across all screen readers, browser engines, and operating systems.

Notably, we did not include JAWS with Edge or JAWS with Firefox, even though they are more commonly used than Voiceover with Safari. We reasoned that since Microsoft Edge uses the same underlying browser engine as Google Chrome, they would likely have the same behavior with screen readers. For budgetary reasons, we prioritized Voiceover with Safari on MacOS over JAWS with Firefox on Windows, since we wanted to have iOS represented.

Note: We felt it was especially important to show the results of Voiceover, since its behavior with SVG and ARIA conforms most closely to the specifications.
